# Objective
- KijaniKiosk, a growing digital platform that connects small retailers to customers across East Africa.
- Documenting Git workflows and DevOps collaboration practices used by the team.

## Milestone: Foundational DevOps Practices

This milestone marks the KijaniKiosk team's first structured effort to establish a shared DevOps foundation. The team has documented and adopted:

- **Git workflow fundamentals** — understanding the working directory, staging area, and commit history as distinct stages of version control.
- **Branching and pull request standards** — rules for keeping PRs small, writing clear titles, and requiring reviews before merging into shared branches.
- **Safe error recovery** — techniques for undoing mistakes without rewriting shared history, protecting the integrity of the repository.
- **Collaborative review culture** — using PRs to capture reasoning, enable automated CI checks, and create a permanent record of decisions.

Together these practices give the KijaniKiosk team a reproducible, team-wide baseline for contributing code safely and transparently as the platform grows across East Africa.
